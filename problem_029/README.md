# Problem 29: Distinct Powers

**Problem source:** [Project Euler Problem 29](https://projecteuler.net/problem=29)

**Problem statement:**

Consider all integer combinations of $a^b$ for $2 \leq a \leq 5$ and $2 \leq b \leq 5$:

$$\begin{align}
2^2=4, &\quad 2^3=8,\quad 2^4=16,\quad 2^5=32 \\
3^2=9, &\quad 3^3=27,\quad 3^4=81,\quad 3^5=243 \\
4^2=16, &\quad 4^3=64,\quad 4^4=256,\quad 4^5=1024 \\
5^2=25, &\quad 5^3=125,\quad 5^4=625,\quad 5^5=3125
\end{align}$$

If these are placed in a sorted sequence with duplicates removed, we get:
$$4, 8, 9, 16, 25, 27, 32, 64, 81, 125, 243, 256, 625, 1024, 3125$$

That gives us **15 distinct terms**.

How many distinct terms are in the sequence generated by $a^b$ for $2 \leq a \leq 100$ and $2 \leq b \leq 100$?

---

## Solution 1: Naive Set

### Approach

- Compute every $a^b$ for $2 \leq a, b \leq 100$.
- Add each result to a Python set, which automatically discards duplicates.
- The answer is the size of the set.

**Reference:** The full Python implementation is available in [`solution_1.py`](solution_1.py).

### Detailed Explanation

- **Step 1: Nested loops**
  Iterate over all pairs $(a, b)$ with $a$ and $b$ in $[2, 100]$.

- **Step 2: Python's arbitrary precision integers**
  Even $100^{100}$ is computed exactly. There is no overflow risk. Python handles these natively.

- **Step 3: Set deduplication**
  `powers.add(a**b)` silently discards any value already present. No explicit duplicate checking is needed.

- **Step 4: Result**
  `len(powers)` gives the count of distinct terms.

- **Efficiency:** This is the least efficient solution in terms of memory. It stores up to 9801 very large integers. However, for this problem size it runs in milliseconds and is by far the easiest to understand. The simplicity of this approach makes it an excellent baseline.

## Solution 2: Tuple Pairs

### Approach

- For each base $a$, compute its primitive root $r$ and power $p$.
- Instead of storing the giant number $a^b = r^{p \cdot b}$, store the lightweight tuple $(r, p \times b)$.
- Collect all tuples in a set and count.

**Reference:** The full Python implementation is available in [`solution_2.py`](solution_2.py).

### Detailed Explanation

- **Step 1: Compute primitive root**
  `r, p = get_primitive_root(a)` reduces each base to its irreducible form.

- **Step 2: Store tuples instead of giant numbers**
  `terms.add((r, p * b))` stores a pair of small integers. The largest value stored is `upper_bound * 6 = 600` (since the largest power for any primitive root in range is 6, for $a = 64 = 2^6$).

- **Step 3: Set deduplication**
  Two tuples are equal if and only if they represent the same power of the same primitive root. So the set correctly deduplicates.

- **Step 4: Result**
  `len(terms)` gives the count of distinct terms.

- **Efficiency:** The key improvement over Solution 1 is that the largest number ever stored or computed is at most 600. No giant integers are involved at all. The set still does the deduplication work, but on tiny data.

## Solution 3: Dictionary of Exponent Sets

### Approach

- Build a dictionary mapping each primitive root $r$ to the set of all exponents it generates.
- For each base $a = r^p$, the exponents generated are $\{p \times b : b \in [2, 100]\}$.
- If $r$ is already in the dictionary, merge the new exponents in; otherwise create a new entry.
- The answer is the total number of exponents across all entries.

**Reference:** The full Python implementation is available in [`solution_3.py`](solution_3.py).

### Detailed Explanation

- **Step 1: Set comprehension for new exponents**
  ```python
  new_exponents = {p * b for b in range(lower_bound, upper_bound + 1)}
  ```
  This generates the set of exponents that base $a$ contributes, using a **set comprehension**. Curly braces `{}` without a colon produce a set; with a colon they produce a dict. Note that `{}` alone always produces an empty dict, so an empty set must be written as `set()`.

- **Step 2: Merge into dictionary**
  ```python
  if r not in root_exponents:
      root_exponents[r] = new_exponents
  else:
      root_exponents[r].update(new_exponents)
  ```
  `.update()` adds all elements from `new_exponents` into the existing set for root $r$. Since it's a set, any exponents already present are silently ignored.

- **Step 3: Result**
  `sum(len(exps) for exps in root_exponents.values())` totals up the distinct exponents across all primitive roots.

- **Efficiency:** Like Solution 2, no giant integers are ever computed. The additional structure of the dictionary makes the duplicate-handling explicit and inspectable (for example, `root_exponents[2]`) directly shows all exponents reachable from any power of 2.

## Solution 4: Inclusion-Exclusion

### Approach

- Avoid sets and hashmaps entirely by counting distinct exponents using pure arithmetic.
- For each primitive root $r$ with associated powers $P = [p_1, p_2, \ldots]$, count $|S_{p_1} \cup S_{p_2} \cup \ldots|$ using the inclusion-exclusion principle, where $S_p = \{p \cdot b : b \in [2, 100]\}$.
- Sum across all primitive roots.

**Reference:** The full Python implementation is available in [`solution_4.py`](solution_4.py).

### Detailed Explanation

- **Step 1: What is $S_p$?**
  For a base $a = r^p$, the exponents it generates are all multiples of $p$ in the range $[p \cdot 2,\ p \cdot 100]$. So:
  $$S_p = \{p \cdot 2,\ p \cdot 3,\ \ldots,\ p \cdot 100\}$$

- **Step 2: Why LCM?**
  The intersection $S_{p_i} \cap S_{p_j}$ consists of numbers that are multiples of both $p_i$ and $p_j$  i.e., multiples of $\text{lcm}(p_i, p_j)$. The LCM is the "sync point" of two sequences: the smallest exponent where two bases produce the same power of $r$, and they sync again every $\text{lcm}$ steps after that.

  For example, $a=4$ ($p=2$) and $a=8$ ($p=3$) both map to root $r=2$. They first collide at exponent $\text{lcm}(2, 3) = 6$, i.e. $4^3 = 8^2 = 2^6$.

  `lcm_list` extends this to an arbitrary subset by chaining:
  $$\text{lcm}(p_1, p_2, p_3) = \text{lcm}(\text{lcm}(p_1, p_2),\ p_3)$$

- **Step 3: Counting multiples in a range**
  The count of multiples of $m$ in $[lower, upper]$ is:
  $$\left\lfloor \frac{upper}{m} \right\rfloor - \left\lfloor \frac{lower - 1}{m} \right\rfloor$$
  This is purely integer arithmetic. No floats, no storage.

  For a subset of powers, the overlapping range where collisions can occur is:
  $$lower = 2 \times \max(\text{subset}), \quad upper = 100 \times \min(\text{subset})$$
  Outside this range, the sequences do not overlap.

- **Step 4: Inclusion-Exclusion**
  To count $|S_{p_1} \cup S_{p_2} \cup \ldots \cup S_{p_k}|$ without double-counting, we use:
  $$\left|\bigcup_{i} S_{p_i}\right| = \sum_i |S_{p_i}| - \sum_{i<j} |S_{p_i} \cap S_{p_j}| + \sum_{i<j<k} |S_{p_i} \cap S_{p_j} \cap S_{p_k}| - \ldots$$
  
  The alternating signs ensure every element is counted exactly once regardless of how many sets contain it. For any element $x$ appearing in exactly $m$ of the sets, the binomial theorem guarantees its net contribution is:
  $$\binom{m}{1} - \binom{m}{2} + \binom{m}{3} - \ldots = 1$$

- **Step 5: Enumerating subsets with `combinations`**
  `combinations(powers, size)` generates all subsets of a given size. The outer loop over `size` from 1 to `len(powers)` produces all non-empty subsets. The sign for each subset of size `size` is $(-1)^{\text{size}}$ (note: `range(len(powers))` is used with `size+1` in the combinations call, so size=0 corresponds to subsets of length 1, giving sign $(-1)^0 = +1$).

- **Step 6: Grouping bases by primitive root (no hashmap)**
  ```python
  root_powers = []
  for a in range(lower_bound, upper_bound + 1):
      r, p = get_primitive_root(a)
      for i, (root, plist) in enumerate(root_powers):
          if root == r:
              root_powers[i] = (root, plist + [p])
              break
      else:
          root_powers.append((r, [p]))
  ```
  A plain list of `(root, powers)` tuples replaces a dictionary. Grouping is done via linear scan. For-else in Python executes the `else` block only if the loop was not exited via `break`.


- **Efficiency:** The largest list of powers for any single primitive root belongs to $r = 2$, which has bases $a = 2, 4, 8, 16, 32, 64$ with powers $[1, 2, 3, 4, 5, 6]$. This gives at most $2^6 = 64$ subsets to evaluate via inclusion-exclusion. No number larger than 600 is ever computed. No sets or hashmaps are used anywhere.

## Mathematical Foundation


### Key Insight: Where Do Duplicates Come From?

The total number of $(a, b)$ pairs is $99 \times 99 = 9801$, yet the answer is only **9183**. There are 618 pairs that produce duplicates. These duplicates arise entirely from bases that share a common **primitive root**.

For example, $4 = 2^2$, so $4^3 = (2^2)^3 = 2^6$, which is the same as $2^6$. Any base that is itself a power of a smaller integer will generate exponents that overlap with those of that smaller base.

This observation drives all four solutions.

### Primitive Root

Every integer $a \geq 2$ can be uniquely written as $a = r^p$, where $r$ is the smallest possible base and $p \geq 1$. We call $r$ the **primitive root** of $a$.

- $64 = 2^6$ → primitive root $r = 2$, power $p = 6$
- $36 = 6^2$ → primitive root $r = 6$, power $p = 2$
- $41$ (prime) → primitive root $r = 41$, power $p = 1$

Once we know the primitive root, the term $a^b = r^{p \cdot b}$ can be represented as the pair $(r, p \times b)$. Two terms are duplicates if and only if they share the same pair.

### Finding the Primitive Root

```python
def get_primitive_root(a):
    for p in range(int(log2(a)), 1, -1):
        r = round(a ** (1/p))
        if r ** p == a:
            return r, p
    return a, 1
```

We search for the largest valid exponent $p$ first (iterating downward from $\lfloor \log_2(a) \rfloor$), because the largest $p$ corresponds to the smallest $r$. Since $r \geq 2$, the maximum possible $p$ satisfies $2^p \leq a$, giving $p \leq \log_2(a)$.

Floating point is used only to produce a candidate integer via `round()`. The exact check `r ** p == a` uses integer arithmetic and is the true source of correctness.

### Why Inclusion-Exclusion Works

For any element $x$ that appears in exactly $m$ of the sets $S_{p_1}, \ldots, S_{p_k}$, the inclusion-exclusion formula counts it:

$$\binom{m}{1} - \binom{m}{2} + \binom{m}{3} - \cdots + (-1)^{m+1}\binom{m}{m}$$

By the binomial theorem applied to $(1-1)^m = 0$:
$$\sum_{j=0}^{m} (-1)^j \binom{m}{j} = 0 \implies \sum_{j=1}^{m} (-1)^{j+1} \binom{m}{j} = 1$$

So every element is counted exactly once, regardless of how many sets contain it.

### Intersection Range Formula

For a subset of powers $T = \{p_{i_1}, p_{i_2}, \ldots\}$, the intersection $\bigcap_{p \in T} S_p$ contains numbers that are:
- Multiples of $\text{lcm}(T)$ (to be in every sequence)
- In every range $[p \cdot 2,\ p \cdot 100]$ simultaneously

The overlapping range is therefore $[2 \cdot \max(T),\ 100 \cdot \min(T)]$, and `count_multiples` counts the multiples of $\text{lcm}(T)$ within it.

## Notes

- The answer is **9183**, meaning 618 of the 9801 pairs $(a, b)$ produce duplicate values.
- All duplicates arise from bases that are perfect powers of a smaller integer. Primes and other "primitive" bases never produce duplicates with each other.
- **Solution 1** is the most readable and is a perfectly valid solution for this problem size. Python's arbitrary-precision integers make it correct without any special handling.
- **Solution 2** introduces the key idea of working in exponent space rather than value space, reducing the data stored from 200-digit integers to pairs of small integers.
- **Solution 3** makes the grouping structure explicit, allowing direct inspection of which exponents each primitive root contributes.
- **Solution 4** is the most mathematically sophisticated, replacing all set operations with the inclusion-exclusion principle. The largest power list (for $r = 2$) has only 6 elements, so at most $2^6 = 64$ subsets need to be evaluated.
